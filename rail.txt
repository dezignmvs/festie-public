🚀 HOW TO DEPLOY THE WHATSAPP BOT TO RAILWAY / RENDER
===================================================

Since the WhatsApp bot requires a persistent browser (Puppeteer), it is best to host it on Railway.app or Render.com instead of Vercel.

---
OPTION A: RAILWAY (Recommended)
---
1. Go to https://railway.app and login with GitHub.
2. Click 'New Project' > 'Deploy from GitHub repo'.
3. Select your 'festnotify' repository.
4. Railway will automatically detect the 'whatsapp' folder if you tell it to. 
   Go to 'Settings' > 'Root Directory' and set it to: /whatsapp
5. Add these 'Variables' (Environment Variables) in Railway:
   - PORT = 3000
6. Add the 'Nixpack' configuration (Railway does this automatically) but ensure it installs Chromium. 
   Railway usually handles this, but if it fails, add this variable:
   - NIXPACKS_PKGS = chromium
7. Once deployed, Railway will give you a public URL (e.g. https://your-app.up.railway.app).
8. Visit that URL to see your new Premium Status Page and scan the QR code!

---
OPTION B: RENDER
---
1. Go to https://render.com and login with GitHub.
2. Click 'New' > 'Web Service'.
3. Connect your GitHub repository.
4. Configure the service:
   - Name: fest-whatsapp-bot
   - Root Directory: whatsapp
   - Environment: Node
   - Build Command: npm install
   - Start Command: npm start
5. Click 'Advanced' > 'Add Environment Variable':
   - PORT = 3000
6. Render's standard environment might not have Chromium. You may need to use a 'Docker' setup on Render if the standard Node setup fails to launch the browser.

---
WHICH REPOSITORY TO USE?
---
You currently have everything in ONE repository: https://github.com/dezignmvs/festnotify.git
- Vercel will host the WEBSITE (the HTML files in the root).
- Railway/Render will host the BOT (using the /whatsapp subfolder).

This is the best way to keep them connected!
